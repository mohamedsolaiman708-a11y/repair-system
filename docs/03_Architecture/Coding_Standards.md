# Coding Standards

## Naming

Classes

PascalCase

Variables

camelCase

Files

snake_case

Constants

camelCase prefixed with k when appropriate.

---

## Widgets

Prefer StatelessWidget.

Use ConsumerWidget when Riverpod is required.

Avoid StatefulWidget unless necessary.

---

## Functions

Small.

Focused.

Descriptive.

---

## Files

Prefer files smaller than 300 lines.

Split large widgets.

---

## Comments

Explain WHY.

Avoid explaining WHAT.

Readable code should explain itself.

---

## Imports

Prefer package imports.

Avoid relative imports when possible.
