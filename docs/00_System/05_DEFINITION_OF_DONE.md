# Definition of Done

A task is NOT complete simply because it works.

A task is considered complete only when all the following conditions are met.

---

## Functional

The feature satisfies all documented business requirements.

---

## Architecture

The implementation follows the project architecture.

No shortcuts.

No temporary solutions.

---

## Code Quality

Readable.

Modular.

Reusable.

Maintainable.

Proper naming.

No duplicated logic.

---

## UI

Responsive.

Consistent.

Easy to understand.

Accessible.

---

## Performance

Efficient database queries.

No unnecessary rebuilds.

Optimized state management.

---

## Security

Authorization respected.

Input validated.

No sensitive data exposed.

---

## Error Handling

Expected failures handled.

Useful error messages displayed.

No application crashes.

---

## Documentation

If implementation changes behavior, documentation must also be updated.

---

## Final Rule

Working code alone is not considered complete.

Quality is part of the definition of done.
